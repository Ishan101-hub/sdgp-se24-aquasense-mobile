# Screenshots Guide

Good screenshots make the repository portfolio-ready.

## Required Screenshots

- Login screen
- Registration screen
- OTP verification
- Home dashboard
- Water status card
- Daily consumption card
- Leakage monitoring screen
- Usage report screen
- Services screen
- Profile screen
- Settings screen
- Theme screen
- Backend `/health` endpoint
- Render live logs
- Firebase deployed app

## Naming Format

```text
docs/screenshots/login.png
docs/screenshots/home-dashboard.png
docs/screenshots/leakage-monitoring.png
docs/screenshots/usage-report.png
docs/screenshots/services.png
docs/screenshots/settings-profile.png
```

## Screenshot Quality Rules

- Crop unnecessary browser/toolbars if possible.
- Use consistent phone frame or browser size.
- Avoid exposing user emails, passwords, API keys, or tokens.
- Use demo/test accounts only.
- Keep image size optimized.
